GoPhish Campaign Setup

Step-by-step guide for running a phishing simulation campaign in an isolated lab.

⚠️ Lab Environment Only

This guide is for isolated lab environments only. Never run GoPhish against real users, real domains, or production email systems. Ensure you have written authorization before proceeding.

1. GoPhish Installation (Kali Linux)

# Download GoPhish (in isolated lab VM)
$ wget https://github.com/gophish/gophish/releases/download/v0.12.1/gophish-v0.12.1-linux-64bit.zip

# Extract
$ unzip gophish-v0.12.1-linux-64bit.zip -d /opt/gophish

# Make executable
$ chmod +x /opt/gophish/gophish

# Start GoPhish
$ cd /opt/gophish && ./gophish

# Admin panel: https://localhost:3333
# Default creds: admin / (check terminal output for temp password)

2. Configure Sending Profile

The sending profile tells GoPhish how to send emails. In your isolated lab, configure it to use the local sandboxed mail server.

SettingValue (Lab)Notes
NameLab SMTPDescriptive name
SMTP Fromit-security@training.localFictional sender address

Username(leave blank)No auth for local SMTP
Ignore Certificate Errors✅ CheckedLab environment only

3. Create Email Template

GoPhish templates support HTML with tracking variables:

<!-- GoPhish Email Template Variables -->
{{.FirstName}} — Recipient's first name
{{.LastName}} — Recipient's last name
{{.Email}} — Recipient's email address
{{.URL}} — Unique tracking URL (auto-generated)
{{.From}} — Sender address
{{.TrackingURL}} — Invisible tracking pixel

<!-- Example: Password Reset Template -->
Subject: ⚠️ URGENT: Your Password Expires in 24 Hours
Body: Dear {{.FirstName}},
Your password expires soon. Click {{.URL}} to reset.
{{.TrackingURL}} <!-- invisible open tracking -->

4. Create Landing Page

The landing page is what users see after clicking the link. For training, this should be the awareness page — not a credential harvester.

✅ Training Best Practice

Configure GoPhish to redirect users to an awareness training page after they click. GoPhish supports importing existing HTML pages or using its built-in editor. Use the Redirect to field to send users to your training page after interaction.

GoPhish landing page options:

  • Import Site: Import HTML from a URL (your training awareness page)
  • Capture Submitted Data: Enable to track who submitted the form (for metrics only)
  • Capture Passwords: ❌ Disable this — not needed for training
  • Redirect to: Set to your awareness training page URL

5. Set Up User Group

Create a group of dummy test accounts in GoPhish:

# Example CSV format for dummy users
First Name,Last Name,Email,Position
Alice,Test,alice@training.local,Analyst
Bob,Test,bob@training.local,Manager
Carol,Test,carol@training.local,Developer
Dave,Test,dave@training.local,HR Staff
# ... (all fictional accounts on sandboxed mail server)

6. Launch Campaign

In GoPhish, create a new campaign combining:

Campaign SettingConfiguration
NameCampaign 1 — Password Reset Baseline
Email TemplatePassword Expiry Alert (from Step 3)
Landing PageTraining Awareness Page (from Step 4)
Sending ProfileLab SMTP (from Step 2)
GroupsLab Test Users (from Step 5)
URLhttp://192.168.56.20 (lab web server IP)
Launch DateSchedule or launch immediately

7. Monitor & Track Results

GoPhish provides a real-time dashboard tracking:

📧
Emails Sent
👁️
Emails Opened
🖱️
Links Clicked
📝
Data Submitted

GoPhish tracks each event with timestamps, allowing you to analyze response patterns, time-to-click, and which users interacted with the simulation.

8. Export Data for Analysis

# GoPhish API — Export campaign results
GET /api/campaigns/{id}/results?api_key=YOUR_API_KEY

# Or use the GoPhish dashboard:
# Campaigns → Select Campaign → Export CSV

# CSV columns exported:
email, first_name, last_name, status, reported, send_date, modified_date

# Status values:
Email Sent → Email Opened → Clicked Link → Submitted Data
💡 SET Toolkit Alternative

The Social Engineering Toolkit (SET) on Kali Linux can also be used for phishing simulations. Launch with sudo setoolkit, choose 1) Social-Engineering Attacks → 2) Website Attack Vectors → 3) Credential Harvester. However, GoPhish is preferred for campaigns because it provides built-in tracking, reporting, and multi-user campaign management.
