🔐 Module 04

Fake Login Concepts

Conceptual analysis of what phishing login pages look like and how to identify them.

⚠️ No Deployable Code

This section provides conceptual analysis only. No functional login pages, credential capture mechanisms, or deployable code is included. The purpose is solely to teach users what to look for.

Purpose of Awareness Pages

In a phishing simulation, when a user clicks a simulated phishing link, they should be redirected to an awareness training page — not a functional credential harvesting page. This page serves to:

  • Educate: Explain that the user just clicked a simulated phishing link
  • Inform: Highlight the red flags they should have noticed
  • Train: Provide actionable steps for future prevention
  • Support: Reassure the user — no data was captured, this is a learning opportunity

Anatomy of a Phishing Login Page

Understanding what makes a phishing page deceptive helps users identify them. Below is a conceptual breakdown of the elements attackers typically include:

🔍 Conceptual Phishing Page Layout — For Analysis Only
🌐 Address Bar: http://login-exampl3.com/signin 🔴 RED FLAG — Misspelled domain, no HTTPS, no padlock
↓
🏢 Company Logo (Copied) + "Sign in to your account" 🟡 Visual mimicry of legitimate branding
↓
📝 Email / Username Input Field + Password Input Field 🔴 Form action points to attacker-controlled server
↓
🔘 "Sign In" Button 🔴 Submits credentials to attacker, not legitimate service
↓
🔗 "Forgot password?" / "Create account" (non-functional links) 🟡 Added for visual legitimacy — often broken or redirect nowhere

Key Red Flags to Teach Users

Red FlagWhat to CheckExample
URL / Domain Is the domain spelled correctly? Is it HTTPS? login-exampl3.com vs login.example.com
SSL Certificate Is there a padlock? Is the certificate valid? Missing padlock or "Not Secure" warning
Visual Quality Are logos blurry? Is the layout slightly off? Low-res logo, misaligned elements
Footer Links Do the "Privacy Policy" / "Terms" links work? Broken links or links to unrelated sites
Form Behavior Does the form submit to the expected domain? Form action URL doesn't match the service
Browser Warnings Is the browser showing any security warnings? Red address bar, phishing warnings

What a Training Awareness Page Should Include

When a user clicks a simulated phishing link in a training exercise, they should land on a page with these elements:

📋 Required Elements

1. A clear "TRAINING SIMULATION" header so the user immediately knows this is an exercise.
2. An explanation of what just happened ("You clicked a simulated phishing link").
3. The specific red flags they should have noticed in the email/link.
4. Actionable tips for identifying phishing in the future.
5. A supportive, non-punitive tone.
6. A link to report the simulated phishing email.
7. Contact information for the security team.

Security Warnings to Include

  • Large, prominent "THIS IS A TRAINING EXERCISE" banner
  • "No credentials were captured or stored" assurance
  • Explanation of the simulation program and its goals
  • Link to the organization's actual security policy
  • Contact details for the security awareness team
  • "This interaction has been logged for training metrics only"
  • Opt-out information if applicable to the training program
✅ Key Principle

A well-designed training awareness page turns a "failure moment" into a powerful learning opportunity. The user walks away more informed, not embarrassed.
